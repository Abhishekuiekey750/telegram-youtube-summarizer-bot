# Install OpenClaw and use it with this bot

OpenClaw is **installed separately** (it is not a Python package). Follow these steps so this Telegram bot uses OpenClaw for summaries.

---

## Step 1: Install Node.js 22+

OpenClaw needs **Node.js 22.12 or higher**.

- **Windows**: Install [Node.js LTS](https://nodejs.org/) (v22 or newer), or use **WSL2** and install Node 22 inside WSL (recommended by OpenClaw on Windows).
- Check:
  ```bash
  node -v
  ```
  You should see `v22.x.x` or higher.

---

## Step 2: Install OpenClaw

In a terminal (or WSL2 on Windows):

```bash
npm install -g openclaw@latest
```

Or without global install:

```bash
npx openclaw@latest gateway
```

Check:

```bash
openclaw --version
```

---

## Step 3: Create OpenClaw config and enable the HTTP API

The bot talks to OpenClaw over an **OpenAI-compatible HTTP API**. You must enable it and set a token.

1. **Create the config directory** (if it doesn’t exist):
   - **Windows**: `%USERPROFILE%\.openclaw\`
   - **Linux/macOS / WSL**: `~/.openclaw/`

2. **Create or edit** `openclaw.json` in that folder.

3. **Put this in the file** (use your own token and port if you change it):

```json
{
  "gateway": {
    "port": 18789,
    "http": {
      "endpoints": {
        "chatCompletions": { "enabled": true }
      }
    },
    "auth": {
      "mode": "token",
      "token": "my-secret-gateway-token-123"
    }
  }
}
```

- Replace `my-secret-gateway-token-123` with a strong secret. You will use this same value in this project’s `.env` as `OPENAI_API_KEY`.
- Default port is `18789`. If you use another port, use it in the URL in Step 5.

4. **Optional**: Run the setup wizard once:
   ```bash
   openclaw onboard --install-daemon
   ```
   You can skip this if you only need the HTTP API for this bot.

---

## Step 4: Start the OpenClaw gateway

In a terminal, run:

```bash
openclaw gateway
```

Leave this running. You should see it listening (e.g. on port 18789).

---

## Step 5: Point this bot at OpenClaw

In **this project’s** folder, open the **`.env`** file (create it from `.env.example` if needed) and add or set these two lines:

```env
OPENAI_BASE_URL=http://127.0.0.1:18789/v1
OPENAI_API_KEY=my-secret-gateway-token-123
```

- Use the **same token** you put in OpenClaw’s `gateway.auth.token`.
- If your gateway uses a different port, change the URL (e.g. `http://127.0.0.1:3000/v1`).

**Important:** Without these two variables, the bot does **not** use OpenClaw; it keeps using the built-in stub and you will see “demo purposes” summaries.

---

## Step 6: Install Python deps and run the bot

In this project folder:

```bash
pip install -r requirements.txt
python -m cmd.bot.main
```

You should see a log like:

`model_factory.openai_enabled` with `base_url=http://127.0.0.1:18789/v1`

Then send a YouTube link to your Telegram bot; the summary will be generated by OpenClaw.

## Summary

| Step | What to do |
|------|------------|
| 1 | Install Node.js 22+ |

| 4 | Run `openclaw gateway` and keep it running |

## If you’re on Windows without WSL

OpenClaw officially recommends **WSL2** on Windows. If you use PowerShell only:

1. Install Node 22 from [nodejs.org](https://nodejs.org/).
2. Run the same commands in PowerShell; if something fails, use WSL2 and run them there instead.

More: [OpenClaw on Windows (WSL2)](https://docs.openclaw.ai/platforms/windows.md)
